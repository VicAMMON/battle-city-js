== Описание игры ==
    В игре существует несколько типов «экранов».
    === Авторизация ===
    === Общий чат ===
    === Premade ===
    === Собственно игра ===
== Общие идеи построения ==
    Вся логика рассчитывается на сервере, клиент имеет право рассчитывать лишь
    заранее определенные последовательности действий (анимацию). Клиент же
    отвечает лишь за обработку событий клавиатуры, и отображение нужных объектов.
    Данные передаваемые между клиентом и сервером можно разбить на два потока: 
    команды управления (от клиента на сервер), данные синхронизации (от сервера на клиент).
== Реализация ==
    Одним из достоинств node.js считается возможность писать серверную и клиентскую
     часть на одном языке. Я попытался использовать эту возможность следующим образом: 
     писать базовый класс с общей функциональностью, и писать для него два рассширения, 
     для клиентской и серверной частей.

== Классы ==
    === TList ===
    Хранит индексированный список объектов, добавляя к каждому объету уникальный id. 
    Генерирует событи при добавлении или удалении объектов.
    === Loggable ===
    Позволяет следить за изменениями объектов в TList, вести лог изменений объектов
    === Player ===
    Игрок. Может:
      * писать сообщения в общий чат или premade,
      * управлять своими игровыми объектами,
      * создавать игру/подключаться к игре/выходить из игры.
== Аспекты ==
    Логгирование событий на сервере, для синхронизации данных с клиентом.
